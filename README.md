# ИС Контроля перемещения мигрантов

Демонстрационный прототип системы отслеживания и анализа перемещений мигрантов на основе геотреков.
Дизайн в стиле МВД России (тёмно-синий + золотой, шрифт PT Sans).

## Быстрый запуск (macOS)

### Вариант 1: Python (встроен в macOS)
```bash
cd migrant-monitor
python3 -m http.server 8080
```
Откройте в браузере: **http://localhost:8080**

### Вариант 2: Node.js
```bash
cd migrant-monitor
npx serve . -l 8080
```
Откройте в браузере: **http://localhost:8080**

> Приложение загружает JSON-данные по HTTP, поэтому простое открытие `index.html` напрямую не сработает из-за CORS. Нужен локальный сервер.

## Возможности прототипа

### Карта
- **Тепловая карта** — визуализация плотности мигрантов по городу
- **Геоточки** — индивидуальные позиции каждого мигранта
- **Кластеры** — автоматическая кластеризация с отображением количества
- **Геозоны** — разрешённые зоны пребывания (пунктирные окружности)
- **Рабочие зоны** — местоположения работодателей и общежитий
- **Маркеры инцидентов** — треугольные маркеры на карте
- **9 подложек** — CartoDB (тёмная/светлая), Яндекс Карта, Яндекс Спутник, ESRI, Google (карта/спутник/рельеф), OpenStreetMap

### Ползунок времени (Timeline)
- Прокрутка по часам за 30 дней (720 временных шагов)
- Кнопки: ◀◀ (−1 день), ◀ (−1 час), ▶ (play/pause), ▶ (+1 час), ▶▶ (+1 день)
- Регулировка скорости воспроизведения: 0.5x — 16x
- Горячие клавиши:
  - `Пробел` — воспроизведение/пауза
  - `←` / `→` — час назад/вперёд
  - `Shift+←` / `Shift+→` — день назад/вперёд
  - `Escape` — закрыть карточку мигранта

### Аналитические кейсы (исходные 4)

#### 1. Отклонение от маршрута
Мигрант должен быть на работе по расписанию (например, 8:00–18:00), но обнаружен в другом месте. Система фиксирует координаты мигранта и ожидаемое место работы.

#### 2. Скопления / кластеры
Необычные скопления мигрантов в определённых точках. 8 предзаданных событий в данных — массовые скопления (20-50 человек) в разное время суток, включая ночные.

#### 3. Нарушение геозон
Мигрант покидает разрешённую зону пребывания (определяется по округу проживания/работы). Система фиксирует выход за пределы и расстояние от зоны.

#### 4. Аномалии по времени
- Ночные перемещения (1:00–5:00) — мигрант активен, когда должен быть дома
- Отсутствие на работе — мигрант остаётся дома в рабочее время

### Новые сценарии мониторинга (5)

#### 5. GPS ≠ Оператор (Сценарий 1)
Расхождение координат GPS мобильного устройства и данных сотового оператора. Указывает на возможную подмену GPS-сигнала или использование VPN/прокси для геолокации. Инциденты генерируются при расхождении более 2 км.

#### 6. Телефон оставлен (Сценарий 3)
Покупка совершена далеко от местоположения телефона. Мигрант мог оставить телефон дома, чтобы скрыть реальное перемещение. Фиксируется при расстоянии более 5 км между покупкой и позицией телефона.

#### 7. Серая занятость (Сценарий 6)
Систематические обеденные покупки далеко от заявленного места работы. Указывает на то, что мигрант фактически работает не там, где декларировано. Анализируются паттерны покупок в 12:00–13:00.

#### 8. Ночное отклонение (Сценарий 14)
Мигрант ночует не по адресу регистрации. Система анализирует координаты в период 23:00–05:00 и сравнивает с зарегистрированным адресом проживания.

#### 9. Инцидент — срез (Сценарий 15)
Снимок окрестности места происшествия. При возникновении криминального инцидента система показывает всех мигрантов в радиусе 500м, с навигацией к месту и времени события.

### Панель инцидентов
- Полный список инцидентов (2220 шт.) с 9 типами
- Вкладки: Инциденты / Сценарии
- Фильтрация по типу: Маршрут / Скопления / Геозоны / Время / GPS≠Опер. / Тел.оставл. / Сер.занят. / Ночн.откл. / Инцидент
- Фильтрация по серьёзности: Высокий / Средний / Низкий
- Клик на инцидент — автоматическая навигация к месту и времени события

### Вкладка «Сценарии»
- Легенда всех сценариев с описанием визуализации
- Список криминальных инцидентов (6 шт.) с навигацией
- Карточка инцидента: название, количество мигрантов в зоне, радиус

### Поиск и карточка мигранта
- Поиск по ID или имени
- Карточка с полной информацией: ФИО, гражданство, занятость, график, геозона, разрешение
- Новые поля: Телефон, IMEI, количество покупок, GPS расхождений, инцидентов
- Отображение дневного маршрута (трек) при выборе мигранта
- Маркеры дома (зелёный) и работы (оранжевый) на карте

## Данные

| Файл | Описание |
|------|----------|
| `data/migrants.json` | 1000 мигрантов с профилями (вкл. телефон, IMEI) |
| `data/tracks/day_YYYY-MM-DD.json` | Почасовые геопозиции за каждый день |
| `data/incidents.json` | 2220 инцидентов девяти типов |
| `data/gps_discrepancies.json` | 1522 записи расхождений GPS/оператор |
| `data/purchases.json` | 4890 записей покупок |
| `data/crime_snapshots.json` | 6 криминальных инцидентов с окрестностями |
| `data/geozones.json` | 5 разрешённых геозон |
| `data/zones.json` | Рабочие зоны и общежития |
| `data/stats.json` | Сводная статистика |

### Масштаб данных
- **1,000 мигрантов** — 6 национальностей, 6 сфер занятости
- **720,000 геоточек** — почасовые позиции за 30 дней (1–30 марта 2026)
- **2,220 инцидентов** — 1532 ночных отклонения, 232 временные аномалии, 179 серая занятость, 136 GPS-расхождений, 71 отклонение от маршрута, 42 нарушения геозон, 14 оставленных телефонов, 8 скоплений, 6 криминальных срезов
- **1,522 GPS-расхождения** — сопоставление координат GPS и оператора
- **4,890 покупок** — карта транзакций с аномалиями
- **6 криминальных срезов** — инциденты с анализом окрестности (R=500м)
- **Регион** — Москва и ближнее Подмосковье

### Регенерация данных
```bash
python3 generate_data.py
```

## Технологии
- **OpenLayers 10.3** — карта, слои, тепловая карта, кластеризация
- **9 базовых карт** — CartoDB, Яндекс, Google, ESRI, OSM
- **Vanilla JS** — без фреймворков, работает из коробки
- **CSS3** — адаптивный тёмный интерфейс в стиле МВД РФ
